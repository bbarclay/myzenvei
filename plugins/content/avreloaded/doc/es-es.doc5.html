Usando listas de Reproducción

Cuando se reproduce contenido local con el reproductor JW Media player, en lugar de usar un solo archivo, usted puede proveerle al reproductor una lista de reproducción en un archivo en formato XML. Tenga en cuenta que si desea proveer una lista, usted debería utilizar la etiqueta auto o una de las variaciones de remote, ya que de esta forma no se adiciona de forma automática ninguna de las extensiones predefinidas. Usando ésta alternativa, en lugar de proveer el URL al video o audio directamente, usted especifica la URL de la lista de reproducción. El reproductor soporta tres formatos usados comúnmente para definir listas de reproducción y así proveer una mayor compatibilidad: XSPF (usado para música), RSS (generalmente usado para Podcasts) y ATOM (de uso regular en Blogs). En el sitio web de Jeroen hay ejemplos tanto de listas de reproducción en formato XSPF y en formato RSS las cuales incluyen la mayoría de las etiquetas soportadas.

Usando listas de reproducción usted puede obtener algunos efectos que no están disponibles cuando se usan URLs que apuntan a videos o audio directamente. Por ejemplo, usted puede pegar una imagen de pre visualición al video la cual será mostrada en cualquier momento en el que el reproductor se detenga. Acá se muestra un ejemplo sencillo (este ejemplo se usa en este sitio para mostrar la imagen de pre visualización en el video que aparece en la página principal):

 
  <playlist version="1" xmlns="http://xspf.orig/ns/0/">
    <title>CG Playlist</title>
    <trackList>
      <track>
        <title>Reloaded</title>
        <location>/images/stories/videos/matrix.flv</location>
        <image>/images/stories/videos/matrix_thumb_small.jpg</image>
      </track>
    </trackList>
  </playlist>
 

Usted debe tener en cuenta que cuando utilice la etiqueta auto especificando una URL que no es absoluta, la lista de reproducción debe estar localizada en la carpeta images/stories/videos aun cuando la lista solo contenga referencias a archivos de audio. Adicionalmente, cuando usted utiliza la extensión .m4v en la etiqueta location, usted debe especificar el tipo del archivo de forma manual para la entrada correspondiente. El tipo de archivo se especifica adicionando el siguiente elemento: <meta rel="type">flv</meta> al final de la entrada correspondiente.

Despliegue de la lista de reproducción

Desplegar la lista de reproducción puede llegar a ser un problema si usted no conoce como el reproductor utiliza los diferentes parámetros. A continuación se explica la lógica: Tanto para el ancho como para el alto existe un segundo parámetro llamados: displaywidth y displayheight. Normalmente a estos dos parámetros no se les asigna valor alguno lo que produce que el reproductor se despliegue en su forma por defecto. Si el valor asignado a uno de estos parámetros es menor la lista de reproducción será desplegada. Si usted escoge displaywidth, la lista de reproducción será desplegada ala derecha de la pantalla de video y encima de la barra de controles. El ancho de la lista de reproducción para este caso es: width - displaywidth. Ahora, si usted usa displayheight el cálculo es un poco más complicado debido a que el alto de la barra de controles debe ser tenido en cuenta. El alto de la barra de controles es de 20px; luego, si usted asigna a displayheight un valor menor que height - 20, la lista de reproducción completa el espacio debajo de la barra de controles. En este caso, el alto de la lista de reproducción es: height - displayheight - 20. Finalmente, existe un caso en el cual no se muestra la lista de reproducción pero se logra un efecto de aparecer/desaparecer cuando el ratón entra y sale del área del video. Para lograr este efecto, se debe asignar a displayheight el mismo valor asignado al alto del video.

Parámetros

Algunos de los Parámetros del reproductor están relacionados con la lista de reproducción: Cuando usted desea que las imágenes definidas en la lista de reproducción se muestren como miniaturas, el valor del parámetro plthumbs debe ser true.
